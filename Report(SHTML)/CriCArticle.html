Abstract

The Crisis Containment Service (CriC) is an accesible web platform which allows persons and organizations to get help in case of, respectively manage catastrophic events such as earthquakes, fires, floods etc. This report describes the application, its implementation and discusses what has the team learned by implementing the project.

Assignement

Develop a Web platform that allows crisis management (eg, earthquakes, fires, floods, etc.) by authorities or organizations. The application will be able to integrate Google Crisis Map and Fusion Tables to create additional layers to provide information of interest regarding events and areas affected. In addition, the Common Alerting Protocol (CAP) will be used to transmit notifications and alerts to the population (affected directly or in the vicinity); Eventually, various social platforms can be used. Using Geolocation APIs and services such as Google Person Finder, the system should be able to automatically provide information of interest in finding a missing person (eg when it is unconscious or injured) and / or related to any shelters and rescue routes.

Implementation

  • The programing language of choice for this application was Java as it is a mature platform with many available frameworks. Also given the relatively low ammount of practical experience regarding web development the team had prior to this project and the academic exposure towards this language made it a natural choice.
  • The main design pattern that is used in the project is the MVC pattern, as was specified in the assignement. The particular implementation that was used is Java Server Faces alongside Spring MVC.
  • The database that was used is a MySQL based one with the ORM (Object-relational mapping) solution employed beeing Hibernate.
